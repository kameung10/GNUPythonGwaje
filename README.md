설명
1. 시작하면 캔버스가 없는, 윈도우 창만 들어있는 객체가 생성됩니다. 이때, 키매니저도 함께 생성됩니다.
2. 키매니저는 어떤 캔버스가 윈도우 창에 pack되어 있는지에 따라 키입력을 다른 캔버스로 보내주는 역할입니다. 기본적으로 메뉴캔버스를 pack해줍니다.
3. 메뉴화면일 때, 화살표를 통해 play, exit 변환이 가능합니다. esc외 다른키는 작동하지 않습니다. (닫는키는 -1을 반환해 창을 닫아줍니다.)
4. 게임화면 객체를 만들면 바탕, 점수, 플레이어, 몹들(3종류)이 생성됩니다.
5. 플레이어의 공격을 통해 생성된 미사일이 몹과 부딪히면 둘 다 사라지고 점수를 얻습니다.
6. 게임화면일 때, 스페이스를 누르면 현재 플레이어의 위치를 인수로 미사일이 생성됩니다.
7. 미사일과 몹이 부딪히는지는 위치비교를 해야하므로 move함수 실행 도중에 return이 불가능합니다. 따라서 yield로 값을 반환합니다. (이 때 함수는 일시정지라고 하네요.)
8. 게임화면에서는 부딪히는지 비교하는 함수를 실행합니다. 내부의 next(...)로 yield 반환값을 불러옵니다.
9. 몹이 None이 아닐 때, 비교하며, 부딪히면 캔버스에서 사라지며 None이 됩니다. (None으로 하지 않으면 미사일이 중간에 사라집니다.)
10. 몹들이 다 사라지면 display함수를 통해 다시 몹들을 생성합니다.
11. 몹도 플레이어에게 공격하게 하고 싶었는데 시간이 부족해 구현하지 못했습니다.
12. (플레이어 미사일은 MissileP, 몹의 미사일은 MissileM으로 생각했는데 만들지는 못했습니다.)
